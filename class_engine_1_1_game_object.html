Detailed Description

Base class for a game object in the game. It's lifecycle can be described as the following:

Construction --> [Next Frame] Begin --> Tick --> [Next Frame] End ^-—|

Objects have a "Began" flag to indicate whether their OnBegin() function got called already. That's needed to give objects exactly one frame time to prepare after constructing them using New<> or NewChildEntity<>. Example:

MyObject obj = New<MyObject>(); obj.Location = new Vector2(0, 0); obj.Rotation = 45.0f; Attach(obj);

In this example, the location and rotation can be set before the object's OnBegin() function will be called, giving it time to initialize and set parameters. OnBegin() will get called next frame, and from then on, it'll also call OnTick().

The same is also done with End. Destroying game objects will cause them to get destroyed on next tick and not the current one.
